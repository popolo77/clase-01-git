# Bitácora - Materia 1: Git, GitHub, Terminal e IA Profesional

## Objetivo de la materia

Aprender a utilizar Git, GitHub y la terminal como herramientas profesionales de control de versiones, documentación, trazabilidad académica y trabajo colaborativo.

Esta bitácora registra el avance clase por clase de la Materia 1 del Programa Complementario de Formación Frontend.

---

## Clase 1 - Git local y primeros commits

En esta clase se trabajó la diferencia entre Git y GitHub, el concepto de repositorio local, working directory, staging area y commit.

### Aprendizajes principales

- Git permite controlar versiones de un proyecto.
- GitHub permite publicar repositorios en la nube.
- `git status` muestra el estado del repositorio.
- `git add` prepara cambios.
- `git commit` guarda cambios en el historial.

### Evidencia generada

- Primer repositorio local.
- Primeros commits.
- Archivos de resumen, objetivos, glosario y reflexión inicial.

---

## Clase 2 - GitHub y repositorio remoto

En esta clase se conectó el repositorio local con GitHub mediante un remoto llamado `origin`.

### Aprendizajes principales

- Diferencia entre repositorio local y remoto.
- Uso de `git remote add origin`.
- Uso de `git push`.
- Creación de un README profesional.

### Evidencia generada

- Repositorio publicado en GitHub.
- Rama principal `main`.
- README inicial del proyecto.

---

## Clase 3 - Git diff y revisión de cambios

En esta clase se aprendió a revisar modificaciones antes de guardarlas en el historial del repositorio.

### Aprendizajes principales

- `git diff` muestra cambios no preparados.
- `git diff --staged` muestra cambios preparados.
- Revisar cambios antes de hacer commit evita errores.

### Evidencia generada

- README actualizado.
- Glosario actualizado.
- Commits con cambios revisados.

---

## Clase 4 - Ramas, git mv y merge

En esta clase se trabajó con una rama de mejora para reorganizar la estructura del repositorio.

### Aprendizajes principales

- Una rama permite trabajar sin afectar directamente `main`.
- `git mv` mueve archivos y prepara el cambio automáticamente.
- `git merge` integra una rama en otra.
- Un merge fast-forward ocurre cuando `main` no recibió otros commits desde que se creó la rama.

### Evidencia generada

- Carpetas `clases/`, `recursos/` y `reflexiones/`.
- Archivos reorganizados.
- Merge hacia `main`.
- Repositorio sincronizado con GitHub.

---

## Clase 5 - Issues y Pull Requests

En esta clase se usó GitHub como herramienta de gestión de tareas.

### Aprendizajes principales

- Un Issue representa una tarea, mejora o problema.
- Un Pull Request permite proponer cambios antes de integrarlos a `main`.
- `Closes #1` vincula un PR con un Issue y puede cerrarlo automáticamente.
- `git pull origin main` trae a la PC los cambios hechos desde GitHub.

### Evidencia generada

- Issue #1.
- Rama `issue-1-politica-uso-ia`.
- Archivo `politica-uso-ia.md`.
- Pull Request.
- Merge desde GitHub.
- Cierre automático del Issue.

---

## Clase 6 - Bitácora técnica y documentación profesional

En esta clase se formalizó la bitácora dentro del repositorio y se actualizó el README para explicar la metodología del programa.

### Aprendizajes principales

- Cada materia tendrá su propio repositorio en GitHub.
- El README funciona como presentación viva del proyecto.
- La bitácora registra el proceso de aprendizaje.
- La documentación también es parte del trabajo profesional.

### Evidencia generada

- Carpeta `bitacora/`.
- Archivo `bitacora-materia-1.md`.
- README actualizado con metodología oficial.
- Repositorio actualizado en GitHub.

---

## Conclusión parcial

La Materia 1 busca demostrar que el aprendizaje no queda solo en teoría, sino en evidencia concreta: archivos, commits, ramas, issues, pull requests, documentación y defensa técnica.

El repositorio funciona como prueba del proceso de formación y como base para el Trabajo Integrador de la Materia 1.
